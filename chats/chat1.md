# 프로젝트: 회의 시간 조율 모바일 웹 서비스 — "모두가 만족하는 시간 찾기"

## 배경 (사용자 리서치 기반)
같은 회사 동료 6명이 다음 주 안에 1시간짜리 회의 시간을 잡아야 한다.
직장인 4명 심층 인터뷰 결과, 진짜 문제는 "빈 시간 찾기"가 아니었다:

1. 사람들은 "물리적으로 되지만 별로인 시간"(점심 직후, 금요일 늦은 오후)을
   말하지 못한다. "투정처럼 보일까 봐", "자기중심적으로 보일까 봐".
2. 반면 "외근이라 안 된다" 같은 정당한 사유는 편하게 말한다.
3. 심지어 약한 선호를 "급한 업무" 같은 거짓 명분으로 위장하기도 한다.
4. 기존 도구(When2meet, 구글시트)는 회원가입·낯선 UI 때문에
   "남들이 안 들어와서" 실패하고, 결국 카톡 단톡방으로 회귀한다.

## 핵심 컨셉: 부정을 긍정으로 뒤집기
기존 도구는 "가능/불가능"만 받는다. 우리는 처음에 "가능하지만 별로"라는
비선호 입력을 고려했지만, 리서치 결과 부정 표현 자체가 부담이므로
프레임을 뒤집었다:

→ 불가능한 시간만 제외하고, 가능한 시간 중 "더 좋은 시간"을 표시하게 한다.
→ 선호를 안 찍은 가능 시간 = 소극적 신호로 자동 역산된다.
→ 사용자는 부정을 말할 필요가 없다. "좋아요"만 찍으면 된다.

## 형태와 진입
- 모바일 웹, 세로 375px 기준. 앱 설치·회원가입·로그인 전부 없음.
- 카카오톡 링크 공유로 진입: 링크 미리보기 카드(OG)가 행동을 유도하고,
  탭하면 카톡 인앱 브라우저에서 0.5초 만에 바로 입력 화면.
- 진입 마찰 제로가 제1원칙.

## 전체 플로우 (7화면)

### [주최자 여정]

1. 회의 만들기
   - 입력: 회의 이름, 소요 시간(기본 1시간), 기간(기본 "다음 주")
   - 주최자는 후보 시간을 고르지 않는다. 기간만 정하면
     시스템이 평일 업무시간 전체를 후보로 연다. (주최자 부담 제거)
   - CTA: "링크 만들기"

2. 카톡 공유 화면
   - 생성된 링크와 카톡 공유 버튼
   - 미리보기 카드가 어떻게 보일지 프리뷰
     (카드 문구 예: "📅 킥오프 회의 시간 정하기 — 탭해서 가능한 시간 알려주세요")

3. 응답 현황 대시보드
   - "6명 중 4명 응답" 실시간 현황, 아바타로 응답자 표시
   - 아직 안 낸 사람에게 "리마인드 링크 다시 공유" 버튼
   - 응답이 다 모이기 전에도 중간 집계 미리보기

4. 결과 / 확정 화면 ★핵심
   - 시간대별 집계: "가능 6 · 하트 4" 식 (부정 카운트는 어디에도 없음)
   - 전원 가능 + 하트 최다 시간이 추천 1위로 자동 정렬, 상단 크게 강조
   - 하트는 익명 집계 (누가 찍었는지 절대 안 보임)
   - CTA: "이 시간으로 확정" → 확정 카드를 다시 카톡에 공유하는 순환 구조

### [참석자 여정]

5. 링크 진입 → 시간 입력 화면 ★★가장 중요, 가장 깊게
   - 진입 즉시 바로 이 화면 (랜딩·설명 페이지 없음, 이름만 살짝 입력)
   - 2단계 입력:
     STEP 1 — 안 되는 시간 제외: 주간 그리드에서 드래그로 쓸어내기
       (외근·선약 등 하드 제약. "월요일 오전 통째로" 같은 큰 단위 제거가 쉽게)
     STEP 2 — 남은 가능 시간 중 "가장 좋은 시간 3개"에 하트 찍기
       (탭 한 번으로 하트가 톡 붙는, 좋아요 수준의 가벼운 손맛.
        3개 제한 = 희소성이 신호를 진짜로 만들고, 우선순위 역할)
   - "하트는 익명으로 집계돼요" 안심 문구
   - CTA: "제출하기"

6. 참석자 완료 화면
   - "선호가 반영됐어요" + 현재 몇 명 응답했는지
   - 확정되면 이 링크에서 확인된다는 안내

7. 확정 화면 (전원 공통)
   - 최종 확정 시간 크게, "우리 중 4명이 이 시간을 좋아했어요" 같은
     긍정적 합의 카피
   - 각자의 참석 구분(필수/선택) 표시
   - 캘린더 추가 버튼

## 디자인 톤 (토스 스타일)
- 미니멀, 여백 넉넉, 둥근 모서리(16px+), 부드러운 그림자
- 토스블루(#3182F6 계열) 포인트 + 뉴트럴 그레이 배경
- 큰 타이포, 화면당 핵심 행동 하나
- 하트/선호는 따뜻한 색(코랄·앰버)으로 긍정적 감정 강조
- 제외된 시간은 조용히 회색 처리 (부정을 시각적으로 강조하지 않음)
- 토스 특유의 친근한 마이크로카피
  (예: "언제가 제일 좋아요?", "딱 3개만 골라주세요", "말 안 해도 알아서 모아드려요")

## 특히 신경 쓸 인터랙션
- STEP 1 드래그 제외: 쓸어내는 손맛, 제외된 영역이 부드럽게 가라앉는 느낌
- STEP 2 하트: 탭 시 하트가 톡 튀어오르는 마이크로 애니메이션,
  3개 다 쓰면 남은 하트 카운트가 자연스럽게 안내
- 결과 화면: 하트가 많은 시간대일수록 시각적으로 따뜻하게 차오르는 히트맵
- 확정 순간: 가벼운 축하 모먼트 (전원 합의의 긍정적 마무리)
